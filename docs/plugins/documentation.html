Plugin API Reference

The full reference is in the repository: docs/PLUGIN-UTILS-REFERENCE.md

That file has every function with its signature, parameters, and a short description. You can give it directly to an AI assistant to generate a plugin — it has enough context to produce working code without needing to read the source.

Globals available inside a plugin

You do not import anything. These are all available directly:

  • window.PluginUtils — the main helper API. Sources, audio, transforms, browser sources, hotkeys, UI, config, modals, popups.
  • window.pluginAPI — file access scoped to your plugin folder (readFile, readConfig).
  • window.obsAPI — raw OBS WebSocket API if you need something PluginUtils does not cover.
  • window.uiHelpers — log to the ROBS console (logInfo, logSuccess, logError).
  • CustomHandlerPlugins.register(plugin) — register a dashboard handler that adds controls to a source row.

Minimal Plugin Template

Copy this into plugin.js to get started. Replace the function body with whatever you want to do.

// plugin.js
(function () {
  window.PluginUtils.registerSidebarButton(
    'MyPlugin',       // plugin name
    'my-plugin-btn',  // unique id
    'Do Thing',       // button label
    async () => {
      try {
        // your code here
        // e.g. await window.PluginUtils.setTextSource('_MyText', 'Hello');
        window.uiHelpers.logSuccess('Done', 'MyPlugin');
      } catch (err) {
        window.uiHelpers.logError(String(err), 'MyPlugin');
      }
    },
    { icon: '▶' }  // optional emoji shown on the button
  );
})();

Giving the Reference to an AI

Open PLUGIN-UTILS-REFERENCE.md and paste it into your AI chat along with a description of what you want the plugin to do. The reference includes all function signatures, what the globals are, and the plugin folder structure — that is enough for a model to write a working plugin in one shot.

Tell the AI:

  • What OBS sources you have and their exact names (including the _ prefix).
  • What you want the button to do when clicked.
  • Whether you need a config file, and what settings it should have.
