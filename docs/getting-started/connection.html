Connection

ROBS connects to OBS over WebSocket. It can run on the same machine as OBS or on a different one entirely. Multiple people can connect to the same OBS session at the same time.

Enable the WebSocket Server in OBS

Go to Tools → WebSocket Server Settings, check Enable WebSocket server, and note the port. Default is 4455. Set a password if you want one.

Set Up the Connection in ROBS

Open the Config panel in ROBS. Enter the host, port, and password, then save. ROBS will connect immediately.

  • Host — localhost if OBS is on the same machine, or the IP of the OBS machine if remote.
  • Port — whatever you set in OBS, default 4455.
  • Password — leave empty if you did not set one.

Profiles

You can save multiple connection profiles. Useful if you connect to different OBS machines. Switch between them in Settings. The Default profile cannot be deleted.

Remote Use

For remote connections, both machines need to be on the same network, or the WebSocket port needs to be accessible from outside. ROBS is just a client, so it does not need to be on the same machine as OBS at all. This is the main reason it exists — you can control a show from a different room or building.

If you are connecting to a remote machine, make sure to use the IP address of the OBS machine instead of localhost. You can either port forward the WebSocket port on the OBS machine, or use a VPN such as Radmin to connect to the same network.

Troubleshooting

  • OBS needs to be running before ROBS connects.
  • Check that the firewall is not blocking the port on the OBS machine.
  • If you changed the port in OBS, make sure ROBS is using the same one.
