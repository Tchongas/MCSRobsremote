Plugins

This is the developer section. It covers how to write a plugin from scratch. If you just want to install one, see Getting Started → Plugins.

Plugin Structure

Each plugin is a folder inside %APPDATA%\robs\plugins. The folder name is the plugin ID. Inside it you need at minimum a plugin.js file. Everything else is optional. 

%APPDATA%\robs\plugins\
└── MyPlugin\
    ├── plugin.js        ← required
    ├── manifest.json    ← optional metadata (name, version, description)
    ├── config.yaml      ← optional settings file
    ├── README.md        ← shown in the Plugins settings tab
    ├── icon.png         ← shown in the Plugins settings tab
    └── data.csv         ← any data files your plugin needs

Two Types of Plugin

There are two things a plugin can do, and they are not mutually exclusive:

  • Sidebar button — a button in the right panel that runs a function when clicked.
  • Dashboard handler — adds controls to a specific source row in the dashboard, matched by source name or type.

Most plugins are sidebar-only. Dashboard handlers are for when you want custom controls that show up inline next to a source.

What the Plugin Gets

When ROBS loads your plugin.js, it runs it in a sandboxed context. The following globals are available inside the plugin:

  • window.obsAPI — the full OBS WebSocket API (scenes, sources, browser, hotkeys, streaming, etc).
  • window.PluginUtils — helper functions so you do not have to call the raw API for common things.
  • window.pluginAPI — file access scoped to your plugin folder, and popup/modal helpers.
  • window.uiHelpers — log to the ROBS console.
  • CustomHandlerPlugins.register() — register a dashboard handler.

See Plugin Documentation for the full API reference.

Adding a Sidebar Button

window.PluginUtils.registerSidebarButton(
  'MyPlugin',   // plugin name
  'my-btn',     // unique button id
  'Do Thing',   // label shown on the button
  async () => {
    await window.PluginUtils.setTextSource('_MyText', 'Hello');
  }
);

Adding a Dashboard Handler

This adds a custom controls panel to any source row matching your canHandle condition.

CustomHandlerPlugins.register({
  name: 'MyPlugin',
  isExternal: true,
  canHandle(sourceKind, sourceName) {
    return sourceName === '_MySpecialSource';
  },
  async execute(options, sourceName, displayName) {
    const btn = document.createElement('button');
    btn.textContent = 'Reset';
    btn.addEventListener('click', async () => {
      await window.PluginUtils.setTextSource(sourceName, '—');
    });
    options.appendChild(btn);
  }
});

Reading Config Files

Put a config.yaml in your plugin folder. Read it with: 

const config = await window.PluginUtils.readYamlConfig({
  myOption: 'default value'
});

Or for a JSON file:

const config = await window.PluginUtils.readJsonConfig('config.json', {
  myOption: 'default value'
});
