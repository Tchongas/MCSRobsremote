Plugins

Plugins are JavaScript files that add buttons to the ROBS sidebar and automate OBS. Drop a plugin into the plugins folder and restart ROBS. The runtime turns exported functions into sidebar buttons.

Plugin Structure

A plugin is a .js file that exports functions. Each exported function becomes a sidebar button. 

function helloPlugin() {
  // Your plugin logic here
}

module.exports = { helloPlugin };

Plugins can use pluginutils to read files, call external APIs, and manipulate OBS sources. The obs client and other helpers are exposed by the runtime.

Package Plugins

A plugin can also be a folder such as myplugin/ with a plugin.js entry point and optional resources like Data.csv, README.md, or configuration files.

Example: RunnerSyncPlugin

The bundled RunnerSyncPlugin syncs two runner slots by reading metadata from a CSV and updating OBS text sources. It demonstrates how to:

  • Read local CSV files from the plugin folder.
  • Fetch external data (MCSR PB/ELO) as fallback.
  • Update text sources and browser URLs in OBS.
  • Add a single sidebar button that runs the full workflow.

Best Practices

  • Keep plugin names descriptive and avoid special characters.
  • Load external resources asynchronously and handle failures gracefully.
  • Update sources efficiently to avoid flooding the OBS WebSocket connection.
  • Store configuration files next to the plugin for easy editing.

Plugin API

The runtime exposes a controlled interface to plugins. Common helpers include:

  • pluginutils.readFile(path) — read a file from the plugin directory.
  • pluginutils.fetchJSON(url) — fetch and parse JSON data.
  • pluginutils.setTextSource(name, value) — update a text source in OBS.
  • pluginutils.setBrowserSourceURL(name, url) — update a browser source URL.
