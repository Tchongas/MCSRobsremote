Writing a Plugin

Plugins are JavaScript files that run inside ROBS and can control OBS, add sidebar buttons, attach controls to source rows, read local files, and open custom UI panels. If you just want to install a plugin someone else made, see Getting Started → Plugins.

Plugin Folder

Create a folder with your plugin name inside %APPDATA%\robs\plugins. The only required file is plugin.js. Everything else is optional. 

%APPDATA%\robs\plugins\
└── MyPlugin\
    ├── plugin.js        ← required
    ├── config.yaml      ← optional settings file
    ├── README.md        ← shown in Settings → Plugins
    ├── icon.png         ← shown in Settings → Plugins
    └── data.csv         ← any other files your plugin needs

What you can do

Plugins can do more than one of these at the same time:

  • Sidebar buttons — add clickable buttons to the right panel that trigger any code you want.
  • Dashboard handlers — attach extra controls to a specific source row when it is expanded.
  • Automation — run sequences of OBS actions in order.
  • Workspaces (BETA) — open a modal or a separate window with a custom HTML interface.
  • Create (BETA) — register a function in Settings → Create to scaffold OBS scenes and sources.

When ROBS loads your plugin, a set of globals are injected into it. You do not import anything. The full reference is in Documentation.
