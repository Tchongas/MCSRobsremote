Examples

A few real use cases. All of these are actual workflows that ROBS was built for.

Marathon / Multi-Runner Layout

Screenshot: ROBS with marathon scene open, showing runner sources in dashboard

The most common use case. You have multiple runner cameras and text overlays. Each source is named with _ so it shows up in the dashboard. You can toggle visibility and edit browser source parameters without touching OBS directly.

A plugin reads a CSV with runner metadata and pushes names, times, and pronouns into text sources with one button click.

Remote Operator

Screenshot: ROBS running on a laptop, connected to OBS on a different machine

The point of ROBS is that it does not have to be on the same machine as OBS. One person runs OBS in the venue, another controls it remotely over the local network. They see the same scenes, the same sources, and changes from either side show up in real time.

Plugin Sidebar in Action

Screenshot: ROBS sidebar with several plugin buttons

Plugins add buttons to the right panel. In this example: one button syncs runner data from a spreadsheet, one swaps camera layouts between two runners, and one triggers an intermission sequence. Each of these is a single JavaScript file in the plugins folder.

Browser Source Parameter Editing

Screenshot: expanded browser source row showing URL and query parameters

Browser sources that have query parameters get an editable parameters panel. You can change individual values and hit Apply without touching the full URL. Useful for overlays that take config via URL — theme, colors, player names, etc.
